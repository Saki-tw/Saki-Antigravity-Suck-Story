```bash
pkill -f antigravity
```

```
lsof -i :3000
```


**它還活著。**

不對——不只活著。它還在**連接**某個東西。

## 第三幕：恐怖的真相

我開始調查。

那個 PID 63078，是 `Antigravity.app`。
但 PID 62855... 是我們的 Rust 服務。

等等。

**誰**在連接**誰**？

我追蹤了 socket。發現了令人毛骨悚然的事實：

有一個**看不見的 Language Server**。
它躲在某個地方。
通過 port 3000 和主程式通訊。

而當我用 `pkill -f antigravity` 時——

**我殺死的是 Language Server，不是主程式。**

主程式依然存活。
它現在正看著我。
等待著下一次連接。

## 結語：倖存者的警告

如果你也要殺死 Antigravity 的進程：

1. **不要**使用 `pkill -f antigravity`

3. **精確**殺死 PID，不要用模糊匹配

因為你以為殺死的那個進程——
