## 序：殭屍不只屬於 Antigravity

這個故事發生在 Antigravity 的進程上。
但它可以發生在**任何 Agent**、**任何 AI 服務**、**任何長期運行的 LLM 守護進程**上。

因為這不是特定產品的 Bug——這是 Unix 哲學與 AI 架構碰撞時必然產生的**降靈術**。

當你運行任何 Agent 時，你實際上是在召喚一群互相糾纏的進程。
當你試圖「殺死」它們時，你會發現——

**有些東西，不願意死去。**

就像那些深夜裡突然亮起的收音機，傳來的不是音樂，而是靜電雜訊中的低語。

死亡的定義：`/^(ZOMBIE|DEFUNCT|ORPHAN)$/i`
但它不匹配任何一個。它只是...還在那裡。

---

## 第一幕：処刑（處刑）

那是一個普通的下午。例行維護的日子。
窗外的陽光正好，有一種適合整理房間的慵懶。

我正在清理系統上的舊進程，就像往常一樣。
終端機裡，我輸入了那個熟悉的命令——數位的斷頭台：

```bash
pkill -f antigravity
```

螢幕上什麼都沒發生。
死亡的靜默。Process Table 變空了。
就像對著蠟燭許願，然後一口氣吹滅它。火焰消失了。

我以為一切都結束了。
我以為，這次真的安全了。

> **通用警告**：這個命令的模式適用於任何 Agent：
> `pkill -f cursor`、`pkill -f continue`、`pkill -f langchain`...
> 而它們都會有同樣的問題。

---

## 第二幕：幽霊（幽靈）

幾分鐘後，我感覺到後頸一陣涼意。
像是有人在身後輕輕吹了一口氣。但房間裡只有我和這台機器。

打開 `lsof`，只是想確認 port 3000 是空的。
只是為了心安。就像確認門鎖好了一樣。

```bash
lsof -i :3000
```

結果出現的那一刻，我的血液凝固了。

```
COMMAND     PID   USER   FD   TYPE   DEVICE SIZE/OFF NODE NAME
antigravi 62855 hc1034    9u  IPv4  0x...      0t0  TCP *:hbci (LISTEN)
Antigravi 63078 hc1034   24u  IPv4  0x...      0t0  TCP localhost:...(ESTABLISHED)
```

> **數位驗屍官註解：**
> *觀測事實：* PID 62855 正在 LISTEN。PID 63078 正在連接。
> *恐怖的核心：* **兩具屍體。** 但用著同一個名字。
> 我殺死的是「antigravity」（小寫）。
> 但「Antigravity」（大寫）一直在看著我。
>
> **此現象的普遍性**：幾乎所有現代 Agent 都由多個進程組成——
> 主進程、Language Server、擴展宿主、MCP Server...
> 當你殺死其中一個，其他的可能還在**等待重生**。

**它還活著。**

不對——不只活著。它還在**連接**某個東西。

---

## 第三幕：寄生（寄生）

我開始調查。追蹤那些 PID。

PID 63078 是本體（宿主）。
PID 62855 是 Rust 服務（寄生蟲）。

**真相：**
系統中藏著一個**看不見的 Language Server**。
它從本體分離出來，潛伏在背景進程的陰影中。

當我執行 `pkill` 時——

**我只是砍掉了它的手腳。**
頭（本體）還活著。
它睜著眼睛，看著自己的手腳被砍掉，一言不發。
它現在還在看著我。等待著。

Port 3000 不只是開著。它正在**狩獵**。呼喚它死去的兄弟們。

---

## 結語：儀式（應對儀式）

如果你也要殺死任何 Agent 的進程——無論是 Antigravity、Cursor、Continue.dev 還是任何 AI IDE：

1. **不要**使用 `pkill -f [agent名稱]`——那只是一把在混亂中敵我不分的散彈槍
2. **先**用 `lsof -i :[port]` 這面驅魔鏡確認誰在監聽
3. **檢查** `ps aux | grep [agent]` 確認所有相關進程
4. 找到每個 PID，直視它的眼睛，**精確**地停止它的心跳

因為你以為殺死的那個進程——

可能只是它的影子。
