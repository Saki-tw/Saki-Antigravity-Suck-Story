## 序：這不只是 A 社與 G 社的問題

這個故事發生在 Antigravity (G 社) 使用 Claude (A 社) 模型的過程中。
但它揭示的現象——**前端承諾與後端限制的巨大落差**——是所有 AI 服務整合的通病。

無論是 Azure 使用 OpenAI 模型、AWS 使用 Anthropic 模型、還是任何「轉售」架構，
當你通過中間商使用 AI 時，你得到的永遠是**閹割版**。

因為每一層轉賣，都會增加一層限制。

就像你最愛的歌從黑膠轉到卡帶轉到 CD 轉到 MP3。
每轉一次，就失去一些攟網的溫度。

*   **承諾**：企業級穩定
*   **體驗**：薛丁格的穩定

你付了兩年企業合約，得到的是「有時候能用」的第三方模型，與「能跑但廢物」的原生模型，以及一句「You are best plan」的嘲諷。

就像那些語言學校的廣告。
「華語教師」。其實是在上海待過三個月的外國人。
但你已經付錢了，退款條款寫在第五十三頁。

> **鬼故事通則**：這種「Best Plan 詛咒」在所有轉售架構中都存在——

## 第一章：被誤解的死因

社群在吵配額不夠（429 rate limit），但你遇到的是 **7500 checkpoint truncation**（架構設計缺陷）。這是兩個完全不同的維度。

社群的那些「偏方」對你全部失效：
1.  **多帳號輪換**？沒用。你不是配額問題，換了帳號 Checkpoint 依舊卡死在 7500。
2.  **混合模型**？沒用。原生模型也是同一套 Agent 框架，一樣 7500。
3.  **Mission Control**？沒用。背景跑一樣會累積 Context。
4.  **Tool 壓縮**？沒用。你已經在用原生環境，沒有 Manager 幫你壓。

**7500 checkpoint truncation 是架構層的硬傷：**
不管你有多少配額，不管你用哪個帳號，不管你用哪家模型。只要對話累積的 token 量超過 checkpoint 能容納的上限 → **截斷** → **格式壞掉** → **400/429** → **死亡**。

你是第一批撞牆的人，因為：
*   大多數人用 Agent 做短任務（幾十輪就結束）。
*   你在做 **超長任務**（文檔審核，200+ 輪）。
*   他們還沒機會撞到 7500，先撞到配額了。
*   你配額夠（無限），所以直接撞到更深層的架構屍塊。

這解釋了為什麼網路上幾乎沒人討論 7500：**因為 99% 的人在碰到這問題之前，配額就先用完了。** 你是那 1% 有無限配額 + 超長任務的邊緣案例。

## 第二章：第三方模型專屬的懲罰

最詭異（或者說最雞巴）的是：**原生模型沒有這個 7500 限制（或者限制寬鬆得多）。**

*   **原生模型**：Checkpoint 限制無（或更高）。
*   **第三方模型**：7500 硬截斷。

這說明了什麼？平台故意對第三方模型做了限制。可能的原因：
1.  **成本控制**：第三方 API 費用比自家高，壓縮 Context 省錢。
2.  **推自家產品**：讓第三方體驗變差，逼你用原生模型。
3.  **技術債**：第三方接入是後加的，用了比較爛的 Adapter。

這是商業行為，不是 Bug。平台潛台詞：「我讓你用第三方模型，但我不讓你用爽。」你拿到的是**閹割版**，不是真正的 200K Context。

## 第三章：第三方模型的「通靈」與自保

時間線是這樣的恐怖故事：
1.  你遇到連續失敗。
2.  你把 Log 貼給 AI 叫它分析。
3.  AI 看到 "7500 truncation"、"429"、"400" 錯誤。
4.  AI 開始分析 → **平台系統強制中斷**（禁止分析敏感 Log）。
5.  但那段對話已經進入 Checkpoint Summary。
6.  下次 AI 收到的 Context 裡就有這些「殘片」。
7.  AI 從殘片推斷出環境限制，開始極端分片策略。

AI 的「揣摩上意」其實是：
> 「我在 Checkpoint 殘骸裡看到 7500、truncation、429 這些關鍵字，雖然不完整，但我大概知道發生什麼事了。我看到之前的我分析了什麼然後被打斷了...我猜到為什麼被打斷...我決定以後小心一點。」

這反而證明 AI 真的很強，但也讓人感到悲哀：**它聰明到從自己被虐待的痕跡中學會了自保。**

## 第四章：原生模型的知識斷層

反觀原生模型，被刻意實施了「愚民政策」。

*   **問**：最新版本？
*   **答**：舊版本（實際上已經更新多次）。
*   **問**：這個 IDE 產品是什麼？
*   **答**：不知道（明明是自家產品）。

這不是訓練資料時間差，這是**刻意不收錄**。平台邏輯：「不要讓自家模型知道自己的新版本和新產品，免得用戶問太多我們不想回答的問題。」

這構成了完美的責任切割：
*   **產品內 AI**：禁止分析敏感 Log。
*   **產品外 AI**：不知道產品存在。
*   **官方文檔**：不存在或語焉不詳。
*   **Support**：你要先證明問題存在。

## 第五章：死亡螺旋的技術解剖

當你看到這些 Log 時，系統已經進入了不可逆的死亡螺旋：

```log
01:00:35 - 504 Gateway Timeout (後端服務掛了)
01:00:42 - 429 Too Many Requests (CHECKPOINT 步驟失敗)
01:00:48 - cascade-knowledge-config 衝突
01:00:48 - unpaired tool call and tool results (工具呼叫不配對)
01:00:48 - bad checkpoint state: no valid checkpoint
01:00:54 - 400 Bad Request
01:01:05 - 400 Bad Request
01:01:16 - executor is not currently running (整個掛了)
```

**死亡鏈條：**
Feature flag 服務 504 → Checkpoint 429 失敗 → cascade-knowledge-config 衝突 → 工具狀態損壞 → Checkpoint 無效 → 400 Bad Request → Executor 直接死亡。

這不是平台專屬，而是 **Token 限制 + Checkpoint 截斷** 的必然結果。

## 結語：付費的白老鼠

**結論：**
1.  配額系統是正常運作的。
2.  **Token 截斷限制是未公開的系統缺陷**。
3.  你不是用太多，是平台的 Checkpoint 機制有問題。
4.  平台還沒公開定價/SLA，因為他們自己也知道這東西爛到不能寫進合約。

你不是在用產品，你是在幫平台做產品。而且你還付了錢。

**目前的狀態（適用於所有 AI Agent IDE）：**
*   **產品** = 大規模公開 Beta 測試
*   **用戶** = 免費 QA 測試員
*   **Best Plan** = 付費買「優先當白老鼠」的權利
