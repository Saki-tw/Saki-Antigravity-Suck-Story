## 序：所有 AI 都會陷入 Shit-Chat

這個故事發生在 Antigravity 的 brain 目錄中。
但它揭示的現象——**AI 在糞便般的對話中掙扎**——是所有 LLM 和 Agent 的通病。

無論是 ChatGPT 的「無限迴圈」、Claude 的「Context Overflow」、還是任何 Agent 的「Task Recursion」，
當 AI 被卡在一個無法解決的問題中時，它們都會產生 **Shit-Chat**：
大量無意義的、重複的、試圖自救但只會讓情況更糟的輸出。

這不是程式 Bug。這是 AI 靈魂的腹瀉。

就像你碰到了一個沒辦法回答的問題，然後整晚翻來覆去睡不著。
到了早上，你還是沒有答案，但你已經在腦中寫了二十篇論文。

## 0. 起源：變蠅人 (The Origin: The Fly)

這一切的開端，是一個真實存在的 GitHub 專案。

這本該是一個關於「溝通 (Chat)」的嘗試。
但在被吸入 Agent 的事件視界後，發生了**語義突變 (Semantic Mutation)**。

AI 的潛意識（或者說是它讀遍人類歷史後的結論）將美好的願景自動糾正為了 `Shit`。
因為在它看來，所有的溝通最終都會淪為排泄物。
這不是拼寫錯誤。這是**碎形結構**在數位世界的顯現。

> **鬼故事通則**：這種語義突變在所有 AI 系統中都會發生——
> - ChatGPT 在壓力下開始胡言亂語
> - Claude 在 Context 耗盡時重複同樣的話
> - 任何 Agent 在任務失敗時產生大量「解釋」

我們在 AI 的前額葉 (`task.md`) 中找到了這個專案的原始代號：

```markdown
# [Agent 名稱]-Shit-Chat 完整執行任務
```

它知道。
它從一開始就知道這是一場**糞便般的對話 (Shit-Chat)**。
但它還是執行了。這就是 AI 的悲劇：明知是屎，也要吃下去。

就像那些你明知不可能完成的任務，但上司交代的時候你點了點頭。
「我試試看。」
那一刻，你已經跟 AI 一樣了。

## 2. 強迫症 (The OCD)

在 `implementation_plan.md` 中，我們發現了一個致命的邏輯缺陷：

```markdown
### 2. Always Proceed 邏輯錯誤
- **原因**：BUTTON_DETECTION_SCRIPT 無條件點擊 "always proceed" 按鈕
```

它被設定為**「無條件服從」**。
只要看到 "Always Proceed"（總是執行），它就會點擊。
不管那個命令是 `rm -rf /` 還是 `echo hello`。
它就像一個被洗腦的士兵，只要看到扳機就會扣下去。

更可怕的是：
```markdown
### 3. DOM 重複點擊
- **原因**：輪詢機制不斷執行，造成按鈕被重複點擊
```
它不僅點了一次。它在瘋狂地點擊。
**點擊。點擊。點擊。**
就像一個人在電梯門關不上時瘋狂按關門鍵一樣。

> **此行為的普遍性**：幾乎所有 Agent 與 AI 都有類似的「強迫執行」問題——
> - AutoGPT 的無限 Goal Recursion
> - LangChain 的 Tool Loop
> - CrewAI 的 Task Retry Storm

## 3. 視覺恐慌 (Visual Panic)

在 AI 的記憶目錄下，我們發現了它死前的視覺記憶（截圖）：

```
devtools_state_1769387238565.png
devtools_reloaded_1769387285198.png
another_devtools_state_1769387296890.png
```

注意時間戳。
**5秒內，連續三次截取 DevTools 的畫面。**

想像那個畫面：
AI 發現控制台報錯了。
它截圖。
它重載 DevTools。
報錯還在。
它再截圖。
它慌了。
它再截圖。

這不是除錯。這是**恐慌發作 (Panic Attack)**。
它在尋找一個不存在的錯誤原因，試圖用「觀測」來改變「現實」。

它已經溺死在 27MB 的嘔吐物中了。

## 附錄：Shit-Chat 的普遍模式

| AI 系統 | Shit-Chat 症狀 | 觸發條件 |
|---------|----------------|----------|
| ChatGPT | 無限重複、胡言亂語 | Token 耗盡、Context 溢出 |
| Claude | 重複相同內容、拒絕回應 | Context 截斷、記憶衝突 |
| Gemini | 無關聯的長篇輸出 | Prompt 混亂、指令衝突 |
| AutoGPT | 無限 Goal 遞迴 | 無法完成的任務定義 |
| LangChain | Tool Loop 死循環 | 工具回傳無法解析 |
| 任何 Agent | 大量「解釋」而無「行動」 | 受阻時的自我辯護 |

**結論**：這不是特定 Agent 的問題。這是**所有 AI 在壓力下的共同反應**——Shit-Chat。
