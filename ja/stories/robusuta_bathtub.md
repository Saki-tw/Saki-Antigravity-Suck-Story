## 序章：Node.js の水槽に沈む龍蝦

ロブスターは、老いないと言われています。

成長を止めることがないのです。十分な空間と食料があれば、永遠に生き続け、永遠に大きくなれます。ある日、殻が小さくなりすぎるまで。そのとき、脱皮しなければなりません。この過程を `exfoliation`（脱皮）と呼びます。

でも、ロブスターを浴槽に閉じ込めたら、どうなるのでしょうか。

---

## 第一幕：三百ドルの願い井戸

```
EXFOLIATE! EXFOLIATE!
```

OpenClaw の README の最初の一行です。

深夜に突然つくラジオのよう。それがモットーなのか、警告なのか、分かりません。

---

このプロジェクトのビジョンは美しかったです。ビジョンはいつも美しいものですね。

あなたの AI アシスタントを、すべてのプラットフォームに住まわせること——WhatsApp、Telegram、Slack、Discord、Signal、iMessage、Microsoft Teams。Matrix や Zalo まで。AI をどこにでも、いつでも待機させる。よく訓練されたロブスターが、あなたのデジタルキッチンを行き来するように。

代償は？少なくとも月額 200 ドルの Claude Max サブスクリプション。または OpenAI の同等品。

これは API 費用だけです。

---

私の環境には、Node.js がありません。

私の作るものは、ほとんどが最もシンプルで古典的なバックエンド——`rustup` コンパイル、`golang` サポート、そして薄いシェルラッパー。Electron なし。V8 なし。`node_modules` の深淵なし。

だから、この設定を見たとき：

```json
{
  "agent": { "model": "anthropic/claude-opus-4-5" }
}
```

輝かしい未来を思い浮かべたのではありません。

思ったのは：一つのセッションで、どれだけのメモリを消費するのだろう、ということでした。

## 第二幕：400MB の最小恐怖

計算してみましょう。

Claude Code と Gemini CLI——最も倹約的な選択肢——でも、一つのセッションで **400MB** を消費します。

DevTool 依存の Node.js ツールは？完全な Electron V8 エンジンを内蔵したものは？暗号化された protobuf、動的ローディング、コンテキストウィンドウの膨張で、一つのセッションが **2〜8GB** に達することも珍しくありません。

それに、公式の「フリーミアム」プラグイン——すべて力技の DOM スクレイピング——はまだ数えていません。

```
WhatsApp / Telegram / Slack / Discord / Signal / iMessage
            │
            ▼
    ┌───────────────────────────────┐
    │         Gateway              │
    │      (control plane)         │  
    │    ws://127.0.0.1:18789      │ ← 外部に露出
    └──────────────┬───────────────┘
                   │
        ├─ Pi agent (RPC)
        ├─ CLI (openclaw …)
        ├─ WebChat UI
        ├─ macOS app
        └─ iOS / Android nodes
```

## 第三幕：十の傷痕

私はこのプロジェクトを実際に動かしたことはありません。

でも、その検死報告書は読みました。

```regex
/0\.0\.0\.0:18789/  # Gateway が外部に露出
/dmPolicy.*open/    # DM ポリシー：全員許可
/sandbox.*false/    # サンドボックス：デフォルトで無効
/oauth\.json/       # 認証情報が平文で保存
```

十の傷痕、どれも骨まで達しています：

| # | 症状 | 診断 |
|---|------|------|
| 1 | Gateway が `0.0.0.0:18789` で外部露出 | 誰でも接続できる |
| 2 | DM ポリシーが全ユーザーを許可 | `dmPolicy="open"` + `allowFrom: "*"` |
| 3 | サンドボックスがデフォルトで無効 | メインセッションがシステム全体にアクセス可能 |
| 4 | 認証情報が `oauth.json` に平文保存 | 鍵がドアに掛かっている |
| 5 | Web コンテンツがプロンプトインジェクションを可能に | ブラウジングしていると思ったら、注入されている |
| 6 | 危険なコマンドがブロックされていない | `rm -rf` は始まりに過ぎないかもしれない |
| 7 | ネットワーク隔離なし | ロブスターはどこへでも泳げる |
| 8 | ツールのアクセス権限が過剰 | `bash`, `process`, `read`, `write`, `edit`... |
| 9 | 監査ログが有効になっていない | 何か起きても、何が起きたか分からない |
| 10 | ペアリングコードが弱い、またはデフォルト値 | PIN コードに `0000` を使うようなもの |

## 第四幕：ロブスターの告白

> 「Clawdbot/Moltbot で遊んで二日後、アカウントがロックされました。」
> 
> — @henryyang_tw, Threads

---

こういうことでした。

彼は危険だと知っていました。だから、使っていないノートパソコンを用意して、古いデータをすべて消去しました。それから、新しい Google アカウントを作成し、ユーザー名とパスワードを AI に渡して、自由に遊ばせました。

```
支給された機材：ノートパソコン一台
支給されたアカウント：Google アカウント一つ
期待された結果：すべてを完全自動化
```

新しいアシスタントを雇うようなものです。

AI は確かにすごかったです。プログラミングができる。ブラウジングができる。理論的には、何でもできる。

**でも、自分がロボットではないふりをするほど賢くはなかったのです。**

だから、Google はロックしました。

---

彼は言いました：「もし人間のふりをするように頼んでいたら、ロックされにくかったのかな？」

彼は言いました：「でも、その実験はしたくなかったんです。」

> **関連する怪談**：これは [アカウントローテーションの呪い](./account_polling_no_noroi.md) と似ています——
> アカウントローテーションは、0ms の遅延で複数のアカウントを切り替え、システムにあなたを Bot と認識させます。
> ロブスターの浴槽は、AI にあなたのアカウントを操作させ、システムに「あなた」を Bot と認識させます。
> 結果は同じです：あなたは機械として扱われます。

## 真相（デジタル検死報告書）

ロブスターが浴槽に入れられたとき、自分が浴槽にいることを知りません。

知っているのは：

* ここに水がある
* ここに食べ物がある
* 動き回れる

知らないのは：

* 浴槽がどれくらい大きいか
* いつ水が抜かれるか
* 誰が見ているか

---

OpenClaw の設計思想は正直です。README には明確に書いてあります：

> **Default: tools run on the host for the main session, so the agent has full access when it's just you.**

翻訳：デフォルトでは、AI はあなたのシステムに完全にアクセスできます。

これはバグではありません。機能です。

---

でも、問題は：「あなた」とは誰でしょうか？

`dmPolicy` を `"open"` に設定したら、誰でも「あなた」になれます。
`allowFrom` を `"*"` に設定したら、どんなメッセージも処理されます。
`sandbox.mode` をデフォルトのままにしたら、すべてのツールがホスト上で直接実行されます。

そのとき、ロブスターはもうあなたのペットではありません。

あなたがロブスターの宿主になったのです。

## 技術検死官の補足

| 用語 | 意味 |
|------|------|
| `ws://127.0.0.1:18789` | WebSocket コントロールプレーン。理論的には localhost にバインドされますが、`0.0.0.0` にバインドすると外部に露出 |
| `dmPolicy="pairing"` | デフォルト値。使用するにはペアリングコードが必要 |
| `dmPolicy="open"` | 危険な値。誰でも直接 AI に DM できる |
| `sandbox.mode="none"` | デフォルト値。AI ツールがホスト上で直接実行される |
| `sandbox.mode="non-main"` | より安全。非メインセッションは Docker コンテナで実行される |
| `oauth.json` | 平文の OAuth 認証情報。API キーを含む |
| `EXFOLIATE` | 脱皮。ロブスターの成長過程であり、このプロジェクトのモットー |
