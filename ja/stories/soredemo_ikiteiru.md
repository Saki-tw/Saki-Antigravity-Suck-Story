## 序章：ゾンビは Antigravity だけのものではありませんの

このお話は Antigravity のプロセスで起こりましたわ。
でも、これは**すべての Agent と AI**、**すべての AI サービス**、**すべての長時間動作する LLM デーモン**で起こりうることですの。

なぜなら、これは特定の製品のバグではなく——Unix の哲学と AI アーキテクチャが衝突した時に必然的に発生する**降霊術**だからですわ。

任意の Agent を実行する時、貴方は実際には互いに絡み合ったプロセスの集まりを召喚していますの。
それらを「殺そう」とした時、貴方は気づくでしょう——

**死にたがらないものがいる**ということに。

窓の外の雨は、そんな悲しいお話を聞いているかのように、静かに降り続けていますわ。

---

## 第一幕：平穏な始まり

それは普通の午後でしたの。

システム上の古いプロセスを整理していましたわ。いつものことですの。ターミナルに、あのお馴染みのコマンドを入力しました——デジタルのギロチン：

```bash
pkill -f antigravity
```

画面には何も起きませんでしたわ。これは普通のこと。`pkill` は本来おしゃべりなコマンドではありませんもの。

私は全てが終わったと思いましたの。
私は安全だと思いましたの。

> **通用警告**：このコマンドパターンはすべての Agent と AI に適用されますわ：
> `pkill -f cursor`、`pkill -f continue`、`pkill -f langchain`...
> そして、すべて同じ問題を抱えていますの。

---

## 第二幕：おかしな兆し

数分後、首筋に冷たいものを感じましたわ。

`lsof` を開きましたの。ただ port 3000 が空いていることを確認するためだけに。

```bash
lsof -i :3000
```

結果を見た瞬間、血が凍りましたわ。

```
COMMAND     PID   USER   FD   TYPE   DEVICE SIZE/OFF NODE NAME
antigravi 62855 hc1034    9u  IPv4  0x...      0t0  TCP *:hbci (LISTEN)
Antigravi 63078 hc1034   24u  IPv4  0x...      0t0  TCP localhost:...(ESTABLISHED)
```

> **デジタル検屍官の注記:**
> *観測事実:* PID 62855 は LISTEN 中。PID 63078 は接続中。
> *恐怖の核心:* **二つの死体。** でも同じ名前を使っていますの。
> 私が殺したのは「antigravity」（小文字）。
> でも「Antigravity」（大文字）はずっと私を見ていたのですわ。
>
> **この現象の普遍性**：ほぼすべての現代の Agent は複数のプロセスから構成されていますの——
> メインプロセス、Language Server、拡張ホスト、MCP Server...
> 一つを殺した時、他のものはまだ**復活を待っている**かもしれませんの。

**まだ生きていますの。**

いいえ——生きているだけではありませんわ。何かに**接続している**のです。

---

## 第三幕：恐ろしい真実

調査を始めましたの。

PID 63078 は本体（宿主）でした。
PID 62855 は Rust サービス（寄生虫）でした。

**真実：**
システムの中に**見えない Language Server** が隠れていましたの。
本体から分離して、バックグラウンドプロセスの影に潜んでいたのですわ。

私が `pkill` を使った時——

**私は手足を切り落としただけでしたの。**
頭（本体）はまだ生きていましたわ。
目を開けたまま、自分の手足が切り落とされるのを見ていて、何も言いませんでしたの。
今も私を見ていますわ。待っているのです。

Port 3000 はただ開いているだけではありませんの。**狩り**をしているのですわ。死んだ兄弟たちを呼んでいるのです。

---

## 結語：生存者の警告

もし貴方も任意の Agent のプロセスを殺そうとするなら——Antigravity でも、Cursor でも、Continue.dev でも、任意の AI IDE でも：

1. `pkill -f [agent名]` は**使わないで**くださいまし——あれは混乱の中で敵味方を区別しない散弾銃ですの
2. **まず** `lsof -i :[port]` という除霊の鏡で誰がリッスンしているか確認してくださいまし
3. **確認して** `ps aux | grep [agent]` ですべての関連プロセスを見つけてくださいまし
4. それぞれの PID を見つけて、目をしっかり見て、**正確に**心臓を止めてくださいまし

貴方が殺したと思ったプロセス——

それはただの影かもしれませんの。
